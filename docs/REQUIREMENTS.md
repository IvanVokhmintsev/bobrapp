# ПРИЛОЖЕНИЕ Б

Реестр функциональных и нефункциональных требований к информационной системе

## FR-PP-01: Управление профессиональным профилем музыканта

### FR-PP-01.1 Отображение страницы профиля артиста

Система должна отображать страницу профессионального профиля артиста пользователю с учетом его прав доступа

Блокирующие критерии:

- Система проверяет права доступа пользователя при открытии страницы профиля
- При наличии прав система отображает страницу профиля
- Профиль отображается в виде структурированных блоков профессиональной информации
- Если часть данных профиля недоступна пользователю, система скрывает эти данные и отображает уведомление об ограничении доступа

Неблокирующие критерии:

- Система отображает подсказки о возможностях взаимодействия с профилем

### FR-PP-01.2 Отображение статуса заполненности профиля

Система должна отображать пользователю статус заполненности блоков профиля

Блоки профиля:

- Фото профиля и название коллектива сверху страницы
- В шапке профиля отображаются кнопки: "связаться с артистом", "добавить в избранное", "просмотреть этап на роудмап"
- На странице отображаются ссылки для прослушивания музыки группы на сторонних ресурсах
- В ленте профиля отображаются публикации группы

Блокирующие критерии:

- Если блок профиля не содержит данных, система отображает его как «не заполнен»
- Система отображает статус заполненности отдельно для каждого блока профиля

Неблокирующие критерии:

- Система отображает рекомендации по заполнению блока профиля

### FR-PP-02 Управление профессиональным профилем (музыкант)

### FR-PP-02.1 Редактирование данных профессионального профиля

Система должна позволять музыканту вносить изменения в данные профессионального профиля

Блоки профиля для заполнения музыкантами:

- Фото профиля
- Название коллектива
- Состав группы и роли участников
- Информация о группе в свободном формате
- Выпущенные альбомы/композиции
- Информация о концертах
- Информация о участиях в шоу
- Посты

Другие блоки профиля:

- Роудмап

Блокирующие критерии:

- Система позволяет музыканту редактировать данные профиля после авторизации
- Система выполняет валидацию обязательных полей и форматов данных перед сохранением
- Если обязательные данные не заполнены или формат данных некорректен, система отображает сообщение об ошибке и не сохраняет изменения
- После успешного сохранения система отображает обновленный профиль

Неблокирующие критерии:

- Система отображает подсказки по заполнению полей профиля
- Система отображает уведомление об успешном сохранении изменений

### FR-PP-02.2 Отмена изменений и восстановление предыдущей версии профиля

Система должна позволять музыканту отменять изменения профиля и восстанавливать последнюю сохраненную версию данных

Блокирующие критерии:

- При отмене изменений система возвращает последнюю сохраненную версию профиля
- Система предотвращает сохранение несохраненных изменений после отмены редактирования

Неблокирующие критерии:

- Система отображает уведомление об отмене изменений
- Система отображает предупреждение о потере несохраненных данных перед отменой изменений

### FR-PP-03 Добавление/обновление информации о проектах в профиле (музыкант)

### FR-PP-03.1 Создание и редактирование проектов в профиле

Система должна позволять музыканту создавать новый проект и редактировать существующую информацию о проекте в профиле (атрибуты "Участники", "Участие в шоу", "Выпущенные альбомы/композиции" , "Информация о концертах")

Блокирующие критерии:

- Система позволяет музыканту создавать новый проект или выбирать существующий для редактирования
- Система отображает данные выбранного проекта для редактирования
- Система сохраняет изменения данных проекта после подтверждения музыкантом
- После сохранения система отображает проект в профиле музыканта

Неблокирующие критерии:

- Система отображает подсказки по заполнению данных проекта
- Система отображает уведомление об успешном сохранении проекта

### FR-PP-03.2 Управление составом участников и историей проекта

Система должна позволять музыканту указывать состав участников проекта, их роли и события проекта

Блокирующие критерии:

- Система позволяет музыканту задавать участников проекта и их роли
- Система позволяет добавлять события и этапы проекта
- Система формирует и сохраняет историю проекта

### FR-PP-04 Управление ценностями и позиционированием (музыкант)

### FR-PP-04.1 Добавление и редактирование ценностей и принципов сотрудничества

Система должна позволять музыканту вводить и обновлять ценности и принципы сотрудничества в профессиональном профиле

Блокирующие критерии:

- Система позволяет музыканту вводить и редактировать ценности и принципы сотрудничества
- Система выполняет валидацию заполнения и ограничений данных перед сохранением
- Если формулировки пустые или некорректные, система требует исправления и не сохраняет изменения
- После сохранения система фиксирует изменения и отображает обновленные ценности в профиле

Неблокирующие критерии:

- Система отображает уведомление об успешном сохранении изменений

### FR-PP-04.2 Отмена изменений блока ценностей и позиционирования

Система должна позволять музыканту отменять изменения блока ценностей и позиционирования

Блокирующие критерии:

- При отмене изменений система не применяет внесенные изменения
- Система сохраняет последнюю актуальную версию данных блока ценностей

Неблокирующие критерии:

- Система отображает уведомление об отмене изменений
- Система отображает предупреждение о потере несохраненных данных

### FR-PP-05 Оценка артиста для сотрудничества (лейбл/продюсер)

### FR-PP-05.1 Отображение информации профиля для оценки артиста

Система должна отображать лейблу или продюсеру профессиональную информацию профиля артиста для оценки уровня и динамики развития

Блокирующие критерии:

- Система отображает ключевые блоки профиля артиста:
- Фото профиля
- Название коллектива
- Состав группы и роли участников
- Информация о группе в свободном формате
- Выпущенные альбомы/композиции
- Информация о концертах
- Информация о участиях в шоу
- Посты
- Система отображает прогресс развития артиста во времени: Роудмап
- Система отображает проекты артиста, включая состав участников и историю проектов

### FR-PP-05.2 Предоставление способов контакта с артистом

Система должна предоставлять лейблу или продюсеру доступные способы связи с артистом на основе настроек профиля(кнопка "связаться с артистом")

Блокирующие критерии:

- Если контактные данные доступны, система отображает доступные способы связи с артистом
- Если контактные данные скрыты, система отображает ограничение доступа
- Если в профиле недостаточно данных для оценки артиста, система отображает информацию о незаполненных блоках профиля

### FR-PP-06 Просмотр контекста пользователя для поддержки

### FR-PP-06.1 Отображение профиля и контекста пользователя для обработки обращения

Система должна отображать сотруднику поддержки профиль пользователя и связанные данные для анализа обращения

Блокирующие критерии:

- Система отображает профиль пользователя при открытии обращения
- Система отображает контекст пользователя, связанный с обращением

### FR-PP-06.2 Фиксация результата обработки обращения

Система должна позволять сотруднику поддержки фиксировать результат обработки обращения пользователя

Блокирующие критерии:

- Система позволяет сотруднику поддержки зафиксировать решение по обращению
- Система сохраняет результат обработки обращения с учетом данных профиля пользователя

Неблокирующие критерии:

- Система отображает уведомление об успешном сохранении результата обработки обращения

### FR-FEED-01 Просмотр ленты активности

### FR-FEED-01.1 Отображение ленты профессиональной активности

Система должна отображать пользователю ленту профессиональной активности в хронологическом виде

Блокирующие критерии:

- Система отображает события активности пользователей в хронологическом порядке
- Система отображает события, связанные с проектами и этапами развития
- Если часть активности скрыта настройками приватности, система отображает только разрешенные события
- Если у пользователя нет активности, система отображает пустую ленту
- Система отображает события с отметкой времени
- Если роль пользователя ограничена, система отображает доступный набор типов событий

### FR-FEED-01.2 Фильтрация и переход к источнику события

Система должна позволять пользователю фильтровать события ленты и переходить к связанным профилям и объектам(кнопки "Не показывать в ленте", "Пожаловаться", "Фильтровать")

кнопка "Фильтровать" позволяет фильтровать по типу публикации

Блокирующие критерии:

- Система позволяет пользователю применять фильтры к событиям ленты
- После применения фильтра система обновляет ленту в соответствии с выбранными параметрами
- Система позволяет пользователю перейти из публикации в профиль артиста

### FR-FEED-02 Публикация профессиональной активности

### FR-FEED-02.1 Фиксация и публикация профессионального события

Система должна фиксировать профессиональные действия музыканта как события и публиковать их в ленте активности (в ленте также должна быть кнопка "Новая публикация")

Блокирующие критерии:

- Система фиксирует профессиональное действие музыканта как событие
- Система автоматически добавляет событие в ленту активности
- Система отображает событие пользователям в соответствии с правами доступа
- Если произошла ошибка сохранения события, система уведомляет пользователя, что событие не опубликовано
- Если действие не относится к профессиональной активности, система не публикует событие
- Если проект, связанный с действием, архивирован, система отображает событие с пометкой о неактивном проекте

### FR-FEED-03 Анализ активности артиста для решения о сотрудничестве

### FR-FEED-03.1 Отображение и анализ активности артиста

Система должна отображать лейблу/продюсеру активность артиста

Блокирующие критерии:

- Система отображает ленту активности артиста в хронологическом виде
- Система позволяет анализировать частоту и типы профессиональных действий артиста
- Система позволяет различать активные и неактивные проекты на основе активности
- Если у артиста отсутствует активность, система отображает пустую ленту
- Если активность устарела, система отображает низкую текущую динамику
- Если часть событий скрыта настройками приватности, система отображает только доступные события
- Если события не подтверждены, система помечает их как неподтвержденные

Неблокирующие критерии:

- Система группирует события по типам активности

### FR-FEED-03.2 Сохранение интересных публикаций и артистов

Система должна позволять лейблу или продюсеру сохранять интересные публикации или артистов для последующего просмотра (кнопка "Добавить в избранное")

Блокирующие критерии:

- Система позволяет сохранять публикации или артистов из ленты активности
- Система добавляет выбранные публикации или артистов в раздел сохраненного контента
- Система позволяет управлять списком сохраненного контента

Неблокирующие критерии:

- Система уведомляет пользователя об успешном сохранении публикации или артиста

### FR-FEED-04 Модерация активности в ленте

### FR-FEED-04.1 Просмотр и оценка события для модерации

Система должна позволять администратору просматривать события активности и оценивать их соответствие правилам платформы

Блокирующие критерии:

- Система отображает администратору детали события активности
- Система позволяет администратору определить, нарушает ли событие правила платформы
- Если событие уже было модерировано, система уведомляет администратора

### FR-FEED-04.2 Применение и фиксация мер модерации

Система должна позволять администратору применять меры модерации и фиксировать результаты проверки события

Блокирующие критерии:

- Система позволяет администратору применять меры модерации (скрытие или удаление публикации, ограничение доступа, предупреждение или блокировка пользователя)
- Система фиксирует действие модерации
- Если требуется дополнительная проверка, система отправляет кейс на эскалацию
- Если пользователь оспаривает решение, система фиксирует апелляцию
- Если произошла ошибка фиксации действия, система уведомляет администратора

Неблокирующие критерии:

- Система уведомляет пользователя о примененных мерах модерации
- Система отображает статус обработки модерационного кейса

### FR-RM-01 Просмотр структуры роудмапа развития

### FR-RM-01.1 Отображение структуры роудмапа и связей этапов

Система должна отображать музыканту структуру роудмапа развития с этапами и связями между ними при переходе из профиля коллектива

Блокирующие критерии:

- Система отображает этапы роудмапа в логическом порядке
- Система отображает связи между этапами роудмапа
- Если роудмап не настроен, система отображает сообщение о недоступности
- Если произошла ошибка загрузки данных, система сообщает о проблеме и предлагает повторить попытку
- Система отображает визуальную схему пути развития

Неблокирующие критерии:

1. Система отображает краткое описание каждого этапа

### FR-RM-01.2 Отображение текущего этапа пользователя в роудмапе

Система должна отображать текущий этап пользователя на роудмапе, если он определен

Блокирующие критерии:

- Если текущий этап пользователя определен, система отображает его на роудмапе
- Отображение текущего этапа не скрывает структуру и связи остальных этапов

### FR-RM-02 Определение текущего этапа и фокуса действий

### FR-RM-02.1 Определение текущего этапа развития пользователя

Система должна определять текущий этап развития пользователя на основе данных о прогрессе

Блокирующие критерии:

- Система определяет текущий этап на основе доступных данных о прогрессе
- Если данных недостаточно, система предлагает заполнить недостающую информацию
- Если произошла ошибка расчета, система уведомляет о временной недоступности определения этапа

Неблокирующие критерии:

1. Система отображает факторы, повлиявшие на определение этапа

### FR-RM-02.2 Отображение задач и приоритетных действий текущего этапа

Система должна отображать задачи и приоритетные действия, связанные с текущим этапом развития

Блокирующие критерии:

- Система отображает задачи, связанные с текущим этапом
- Система отображает приоритетные действия для текущего этапа
- Если этап устарел, система предлагает пересчитать статус

Неблокирующие критерии:

- Система отображает рекомендации по выполнению задач
- Система отображает прогресс выполнения задач

### FR-RM-03 Оценка прогресса и критериев этапа

### FR-RM-03.1 Отображение критериев текущего этапа

Система должна отображать измеримые критерии текущего этапа развития пользователя

Блокирующие критерии:

- Система отображает список критериев текущего этапа
- Система отображает статус выполнения каждого критерия
- Если критерии временно недоступны, система сообщает об этом

Неблокирующие критерии:

- Система отображает пояснения к критериям
- Система отображает дату последнего обновления критериев

### FR-RM-03.2 Сопоставление прогресса пользователя с критериями этапа

Система должна сопоставлять данные прогресса пользователя с критериями этапа и отображать степень выполнения

Блокирующие критерии:

- Система определяет выполненные и невыполненные критерии
- Если данные прогресса отсутствуют, система отображает неполную оценку
- Система отображает, какие критерии необходимо выполнить для перехода на следующий этап

Неблокирующие критерии:

- Система отображает процент выполнения этапа

### FR-RM-04 Использование гайдов и чек-листов по этапу

Неблокирующие критерии:

- Система позволяет выбрать формат файла выгрузки
- Система отображает уведомление об успешном формировании файла

Возможность восстановить аккаунт при потере пароля, логина

Моментальная обработка платежей и изменение статуса подписки

- Двойная оплата отсутствует в 99% случаев
