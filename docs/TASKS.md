- [ ] Реализовать регистрацию

- [ ] Возвращать JWT после успешного входа
- [ ] Добавить middleware авторизации
- [ ] Добавить базовую обработку ошибок

Endpoints:

- `POST /auth/register`
- `POST /auth/login`
- `GET /auth/me`

Expected result:

- пользователь может зарегистрироваться;
- пользователь может войти;
- защищенные endpoints доступны только авторизованному пользователю.

## Stage 4 — Users and Roles

- [ ] Добавить роли `musician` и `label`

Expected result:

- пользователь имеет роль;
- роль влияет на сценарий после регистрации.

## Stage 5 — Musician Onboarding

- [ ] Добавить экранные данные для онбординга
- [ ] Реализовать сохранение уровня музыканта
- [ ] Поддержать 4 уровня:
  - `nothing`
  - `beginner`
  - `advanced`
  - `professional`
- [ ] Добавить endpoint для обновления уровня

Possible endpoints:

- `POST /onboarding/musician`
- `GET /profile/me`

Expected result:

- музыкант после регистрации может выбрать уровень;
- уровень сохраняется в профиле.

- [ ] Реализовать получение ленты
- [ ] Реализовать создание поста
- [ ] Реализовать лайк поста
- [ ] Добавить типы постов:
  - `professional`
  - `roadmap`
- [ ] Добавить переход к профилю автора на уровне данных

Endpoints:

- `GET /posts`

Expected result:

- пользователь видит ленту;
- музыкант может создавать посты;
- пользователь может лайкать посты.

## Stage 7 — Roadmap

- [ ] Создать модель RoadmapStep
- [ ] Создать модель UserRoadmapProgress
- [ ] Добавить seed-данные этапов roadmap
- [ ] Реализовать получение roadmap пользователя
- [ ] Добавить статусы этапов:
  - `locked`
  - `available`
  - `completed`
- [ ] Реализовать открытие следующего этапа после завершения текущего

Endpoints:

- `GET /roadmap`
- `GET /roadmap/:stepId`
- `POST /roadmap/:stepId/complete`

Expected result:

- пользователь видит roadmap;
- этапы имеют статусы;
- после завершения этапа открывается следующий.
