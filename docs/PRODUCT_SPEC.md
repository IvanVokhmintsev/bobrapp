# Product Specification — Music Roadmap App

## 1. Описание проекта

Music Roadmap App — приложение для начинающих и развивающихся музыкантов, которое помогает пользователю пройти путь профессионального роста через персонализированный roadmap, социальную ленту и профиль достижений.

Основная идея MVP: пользователь регистрируется, выбирает свою роль, после чего попадает в приложение с тремя ключевыми разделами:

1. Социальная лента
2. Roadmap развития музыканта
3. Профиль пользователя

Продукт ориентирован на музыкантов нового поколения, которые создают музыку самостоятельно, работают в DAW, развиваются через DIY-подход и хотят понимать, какие шаги им нужно делать дальше для профессионального роста.

## 2. Цель MVP

Цель MVP — реализовать core-функционал приложения, который позволяет:

- зарегистрироваться и войти в приложение;
- выбрать тип пользователя: музыкант или лейбл;
- для музыканта пройти первичный опрос по определению уровня подготовки;
- получить доступ к главному интерфейсу приложения: социальная сеть, roadmap, профиль;
- просматривать социальную ленту с постами других музыкантов;
- создавать собственные посты;
- поэтапно проходить roadmap развития;
- проходить простые уроки внутри roadmap;
- отвечать на короткие вопросы после теоретического блока;
- видеть статус этапов roadmap;
- видеть прогресс в roadmap и профиле;
- дополнять профиль информацией о себе как о музыканте;
- добавлять музыку, события и профессиональные достижения;
- видеть достижения и прогресс в профиле.

### Не входит в первую версию MVP

Следующие функции выносятся за рамки первой технической итерации:

- платная подписка для лейблов;
- trial period для лейблов;
- платежи;
- продвижение постов и профилей;
- расширенный поиск для лейблов;
- матчинг лейблов и музыкантов;
- аналитика для лейблов;
- сложная рекомендательная система;
- полноценная медиаплатформа для хранения аудио и видео.

## 3. Основные пользовательские роли

### 3.1. Musician

Основная роль для MVP.

Музыкант может:

- зарегистрироваться;
- выбрать роль `musician`;
- пройти опрос по уровню подготовки;
- просматривать ленту;
- публиковать посты;
- проходить roadmap;
- видеть свой прогресс;
- редактировать профиль;
- добавлять профессиональные достижения.

### 3.2. Label

В MVP роль лейбла реализуется минимально.

Лейбл может:

- зарегистрироваться;
- выбрать роль `label`;
- просматривать социальную ленту;
- открывать профили музыкантов.

На этапе MVP лейбл не получает расширенный поиск, подписку, матчинг и аналитику.

## 4. Основной пользовательский сценарий

### 4.1. Регистрация и вход

Пользователь открывает приложение и может:

- создать аккаунт;
- войти в существующий аккаунт.

При регистрации пользователь указывает базовые данные:

- имя;
- email;
- пароль;
- роль: `musician` / `label`.

После регистрации пользователь попадает на онбординг.

Email verification можно добавить после базовой авторизации. В первой технической итерации достаточно реализовать регистрацию и вход без обязательного подтверждения email.

### 4.2. Онбординг музыканта

Если пользователь выбрал роль `musician`, он проходит короткий опрос.

Цель опроса — определить уровень подготовки пользователя.

В MVP нужно поддержать 4 уровня:

1. `nothing` — пользователь почти ничего не знает.
2. `beginner` — новичок, уже пробовал что-то делать.
3. `advanced` — продвинутый пользователь, регулярно занимается музыкой.
4. `professional` — профессиональный музыкант с опытом релизов, выступлений или коммерческих проектов.

После прохождения опроса система сохраняет уровень пользователя и использует его для отображения roadmap.

### 4.3. Главный интерфейс приложения

После входа пользователь попадает в приложение с нижней навигацией из трех разделов:

1. Социальная сеть
2. Roadmap
3. Профиль

## 5. Основные экраны MVP

### 5.1. Auth Screens

#### Sign Up Screen

Экран регистрации.

Поля:

- name;
- email;
- password;
- role: `musician` / `label`.

Действия:

- создать аккаунт;
- перейти на экран входа.

#### Sign In Screen

Экран входа.

Поля:

- email;
- password.

Действия:

- войти в приложение;
- перейти на экран регистрации.

### 5.2. Onboarding Screen

Экран показывается только музыкантам после регистрации.

Пользователь отвечает на короткий вопрос:

> Как ты оцениваешь свой текущий уровень в музыке?

Варианты:

- Я почти ничего не знаю;
- Я новичок;
- Я уже уверенно занимаюсь музыкой;
- Я профессионально занимаюсь музыкой.

После выбора уровень сохраняется в профиле пользователя.

### 5.3. Feed Screen

Главная страница приложения.

Это социальная лента, где отображаются посты музыкантов.

Посты могут быть двух типов:

#### 1. Обычные профессиональные посты

Примеры:

- выступление;
- релиз трека;
- запись альбома;
- участие в проекте;
- коллаборация;
- новость о развитии.

#### 2. Roadmap-посты

Примеры:

- пользователь прошел этап roadmap;
- пользователь получил достижение;
- пользователь повысил уровень;
- пользователь завершил практическое задание.

Карточка поста должна содержать:

- имя автора;
- аватар автора;
- текст поста;
- тип поста;
- дату публикации;
- количество лайков;
- кнопку лайка;
- возможность перейти в профиль автора.

В MVP можно использовать простую серверную модель постов без сложных рекомендаций.

### 5.4. Roadmap Screen

Экран roadmap — ключевой экран продукта.

Логика должна напоминать Duolingo:

- есть последовательность этапов;
- пользователь видит свой текущий этап;
- доступные этапы можно открыть;
- закрытые этапы заблокированы;
- после прохождения этапа пользователь продвигается дальше;
- за прохождение этапов растет прогресс.

Roadmap должен отображать:

- список этапов;
- статус каждого этапа:
  - `locked`;
  - `available`;
  - `completed`;
- название этапа;
- краткое описание;
- кнопку для открытия материала.

Пример этапов для MVP:

1. Знакомство с музыкальным путем
2. Основы DAW
3. Создание первого музыкального фрагмента
4. Основы структуры трека
5. Запись вокала или партии
6. Базовое сведение
7. Подготовка первого релиза
8. Публикация достижения в профиль

### 5.5. Roadmap Lesson Screen

При нажатии на этап открывается экран материала.

В первой версии урок roadmap содержит:

- название этапа;
- краткий текстовый обучающий материал;
- чек-лист действий;
- короткий quiz на понимание материала;
- кнопку завершения этапа.

Quiz-логика MVP:

- пользователь отвечает на несколько вопросов;
- если пользователь ошибся, система может показать только вопросы с ошибками;
- этап считается завершенным после правильного прохождения всех вопросов.

После завершения:

- этап получает статус `completed`;
- следующий этап открывается;
- пользователю начисляются очки;
- достижение появляется в профиле;
- при необходимости создается roadmap-пост в ленте.

Изображения, видео, схемы и таблицы можно добавить позже как расширение формата уроков.

### 5.6. Profile Screen

Профиль пользователя показывает информацию о музыканте и его развитии.

Профиль должен содержать:

- имя пользователя;
- роль;
- уровень подготовки;
- краткое описание;
- аватар;
- roadmap progress;
- рейтинг/очки;
- достижения в roadmap;
- профессиональные достижения.

#### Roadmap achievements

Примеры:

- прошел этап "Основы DAW";
- завершил первый трек;
- прошел 3 этапа roadmap;
- получил новый уровень.

#### Professional achievements

Примеры:

- выступления;
- записанные альбомы;
- синглы;
- коллаборации;
- релизы;
- участие в проектах.

## 7. Development Notes

Для первой версии важно реализовать core-функционал без усложнения архитектуры.

Backend первой версии реализуется как отдельное API на Node.js, Fastify и TypeScript.

Приоритет:

1. Auth
2. Roles
3. Musician onboarding
4. Feed
5. Roadmap
6. Profile
7. Progress and achievements

Все функции, связанные с монетизацией, сложной аналитикой, рекомендациями и продвинутой ролью лейбла, должны быть вынесены в будущие версии.
